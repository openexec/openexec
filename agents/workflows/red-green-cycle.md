<instructions>
Execute a single red-green test cycle.
</instructions>

<process>
1. Write a failing test that describes expected behavior (RED)
   - Test should be specific and focused
   - Test should fail for the right reason
2. Verify the test fails as expected
3. Write minimal code to make the test pass (GREEN)
   - Only enough code to pass the test
   - Resist urge to add more functionality
4. Verify all tests pass
5. Validate against acceptance criteria
6. Decide: refactor now or continue to next cycle
</process>
