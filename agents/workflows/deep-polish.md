<instructions>
Phase 2: Simplify, apply clean patterns, make code DRY.
</instructions>

<process>
1. Scan code for improvement opportunities:
   - Duplicated code — extract and reuse
   - Long methods — break into focused pieces
   - Complex conditionals — simplify or extract
   - Unclear naming — rename for clarity
   - Code smells — apply appropriate refactoring
2. For each opportunity:
   a. Apply refactoring in small, safe steps
   b. Run tests after each change
   c. If tests fail, revert immediately
3. Consider design patterns where applicable
4. Ensure changes improve readability
5. Document any tests added (only additions allowed)
</process>
